## 📲 Quick Start Deploy (5 Steps)

### Step 1: Deploy Backend to Railway (15 min)

```bash
# 1. Go to https://railway.app and sign up
# 2. Create new project
# 3. Connect GitHub repository: ZbienVC/travel-expense-tracker
# 4. Add PostgreSQL database service
# 5. Set environment variables:

NODE_ENV=production
PORT=3000
API_URL=https://travel-expense-tracker.up.railway.app
DATABASE_URL=postgresql://... (auto-generated by Railway)
GOOGLE_VISION_KEY=... (add your key)
GOOGLE_MAPS_API_KEY=... (add your key)

# 6. Deploy! Railway will auto-deploy on push
# 7. Get your URL from Railway dashboard
```

### Step 2: Deploy Frontend to Expo (10 min)

```bash
# 1. Go to https://expo.dev and sign up
# 2. Install EAS CLI: npm install -g eas-cli
# 3. Login: eas login
# 4. Update app.json with Railway URL
# 5. Publish to Expo:

eas publish --release-channel production

# 6. Or build APK for Android:

eas build --platform android --profile development

# 7. Get your Expo URL or APK download link
```

### Step 3: Generate QR Code (optional)

```bash
# Install qrcode tool
npm install -g qrcode

# Generate QR code for Expo app
qrcode "https://expo.dev/@zach/travel-expense-tracker" -o qr-code.png

# Scan with Expo Go app to test instantly
```

### Step 4: Test Backend

```bash
# Check backend health
curl https://travel-expense-tracker.up.railway.app/health

# Expected response:
# {"status":"ok","timestamp":"2026-03-14T..."}
```

### Step 5: Test Mobile App

```
1. Open Expo Go app on phone
2. Scan QR code
3. Create test trip
4. Add expenses
5. Test receipt scanner
6. View analytics
```

## 🐛 Common Issues & Fixes

### Backend Issues

| Issue | Fix |
|-------|-----|
| Build fails | Check env vars in Railway Variables |
| Database connection error | Verify DATABASE_URL format and PostgreSQL is running |
| Google Vision API error | Ensure key is valid and API is enabled in GCP |
| CORS errors | Verify `cors()` middleware is in place |
| Port conflict | Ensure PORT=3000 is set |

### Frontend Issues

| Issue | Fix |
|-------|-----|
| Can't connect to backend | Check API URL in app.json matches Railway URL |
| Build fails | Run `npm install` and `eas build` again |
| QR code not working | Regenerate with latest Expo publish |
| App crashes on startup | Check logs in Expo dashboard |
| Receipt scanning fails | Verify image quality and Google Vision config |

### Database Issues

| Issue | Fix |
|-------|-----|
| Migrations fail | Run `npx prisma migrate deploy` after Railway deploy |
| Data not persisting | Check DATABASE_URL and connection |
| Query timeouts | Check PostgreSQL service status in Railway |
